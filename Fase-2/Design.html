Propósito

El propósitos de este sitio es realizar el estudio de algunos conceptos básicos de las CSS, para facilitar su comprensión y uso, principalmente orientando hacia el fortalecimiento de competencias que se requieren para en el adecuado diseño de sitios web, y que estos puedan estar a la altura de las exigencias actuales.

Sintaxis CSS3

Según Recio (2016) Las CSS (coding style sheets) u hojas de estilo en cascada son los archivos responsables de defnir la apariencia de una página web. Facilitan la gestión y la apariencia corporativa de un sitio web, ya que todas las páginas de un sitio pueden compartir la misma hoja de estilos y los cambios de apariencia quedan centralizados es estos archivos. Con el uso de CSS3 se siguien ciertas reglas, las cuales se cuentan de los siguientes elementos:

  • Selectores: indica a qué elementos se va a aplicar la regla. Pueden escribirse varios selectores para la misma regla separados por comas
  • Lista de Declaraciones: los estilos que se van a aplicar a los elementos que cumplen la regla.

    • Selectores CSS

    Según Recio (2016) los estilos que se van a aplicar a los elementos que cumplen la regla, los más usados son:

  • Selector Universal: Se defne con * y se aplica a todos los elementos de la página.
  • Selector de etiqueta HTML: Indica el estilo que se va a un elemento o etiqueta concreto.
  • Selector de clase: Podemos aplicar una regla a todos los elementos que sean de una determinada clase
  • Selector de identificador: Permiten aplicar un estilo único al único elemento de la página que tiene un determinado identifcador (especicado con el atributo id). El selector se especifíca con # y el nombre del identicador.
  • Selectores descendentes y adyacentes: Permiten seleccionar elementos contenidos dentro de otros de acuerdo a la estructura en árbol del DOM.
  • Selectores de atributos: Se utilizan para seleccionar elementos en función de sus atributos

  • Interacción Humano Ordenador

    JavaScript es la tecnología que permite la interacción con el usuario. La estandarización de JavaScript con HTML5 en todos los navegadores nos permite crear innumerables tipos de aplicaciones web que pueden ejecutarse en cualquier plataforma (tanto de escritorio como móvil).

    Estas aplicaciones se ejecutan en el cliente, es decir, en el navegador de los usuarios, mientras se mantiene la comunicación asíncrona con el servidor en segundo plano. De esta forma es posible realizar cambios sobre las páginas sin necesidad de recargarlas, mejorando la interactividad, velocidad y usabilidad en las aplicaciones.
